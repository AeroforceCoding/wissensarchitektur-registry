# Dokumentation

Dieser Bereich ist der spätere Andockpunkt für übergreifende Dokumentation, Architekturentscheidungen, Betriebsmodelle und Konzepte.

Aktuell bleibt die bestehende Dokumentation in den Bereichs-READMEs und im Agentensystem maßgeblich. Dieser Ordner schafft nur eine klare Stelle für spätere, größere Dokumentationspakete.

## Vorgesehene Inhalte

Hier können später unter anderem entstehen:

- Architekturentscheidungen,
- Zielbilder,
- Betriebsmodelle,
- Export- und Integrationskonzepte,
- RAG- und lokale-LLM-Konzepte,
- Mehr-Agenten-Konzepte,
- menschlich lesbare Systemübersichten.

## Formatregel

Dokumentation wird bevorzugt als Markdown gepflegt. Sie erklärt Daten, Regeln, Prompts, Automatisierung und Laufzeitumgebung, ersetzt diese aber nicht.
